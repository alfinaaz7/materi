Pengertian GIT dan GITHUB
GIT adalah sebuah sistem pengontrol versi (Version Control System) yang diciptakan oleh Linus Torvalds pada tahun 2005. Git berfungsi untuk melacak dan mengelola perubahan pada kode sumber dalam proyek perangkat lunak secara terdistribusi, sehingga setiap programmer memiliki salinan lengkap dari database Git. Dengan Git, programmer dapat bekerja secara kolaboratif, melacak perubahan kode, kembali ke versi sebelumnya jika terjadi kesalahan, serta mengelola cabang dan penggabungan kode dengan efisien. Sementara itu, GitHub adalah sebuah platform pengembangan perangkat lunak berbasis cloud yang memanfaatkan Git sebagai sistem pengontrol versinya. GitHub memungkinkan para developer untuk menyimpan, mengelola, dan berkolaborasi dalam proyek perangkat lunak secara online. Selain sebagai tempat penyimpanan kode, GitHub juga berfungsi sebagai platform sosial bagi developer untuk berbagi karya, membangun jaringan, dan mengelola proyek dengan fitur-fitur seperti repository, commit, branch, merge, dan pull request. 
BEKERJA DENGAN GITHUB
Bekerja dengan GitHub melibatkan beberapa langkah utama yang memudahkan kolaborasi dan pengelolaan proyek perangkat lunak secara online. Pertama, pengguna harus membuat akun GitHub dan login ke platform tersebut. Setelah itu, langkah berikutnya adalah membuat repository, yaitu tempat penyimpanan proyek yang bisa bersifat publik atau privat sesuai kebutuhan. Di dalam repository, pengguna dapat membuat file kode baru, mengeditnya, dan melakukan commit untuk merekam setiap perubahan yang dilakukan. GitHub juga memungkinkan pembuatan branch atau cabang, yang memungkinkan pengembangan fitur baru secara terpisah tanpa mengganggu kode utama. Setelah perubahan selesai, pengguna dapat membuka pull request untuk mendiskusikan dan mereview kode bersama tim sebelum menggabungkannya ke branch utama. Proses ini mendukung kolaborasi efektif antar developer, memudahkan pelacakan perubahan, dan menjaga stabilitas proyek. Selain itu, GitHub menyediakan fitur sosial untuk berdiskusi, berbagi, dan belajar bersama komunitas developer secara luas.
BRANCH
Branch dalam Git adalah sebuah cabang atau percabangan dari sebuah repository yang memungkinkan pengembang untuk membuat perubahan kode secara terpisah dari branch utama (biasanya disebut main atau master). Dengan menggunakan branch, pengembang bisa mengerjakan fitur baru, perbaikan bug, atau eksperimen tanpa mengganggu kode yang sudah stabil di branch utama. Setiap branch berfungsi sebagai lingkungan terpisah yang memungkinkan perubahan dilakukan secara paralel dan terorganisir. Setelah pekerjaan di branch selesai dan sudah diuji, branch tersebut dapat digabungkan kembali ke branch utama menggunakan proses merge. Penggunaan branch sangat membantu dalam kolaborasi tim karena memungkinkan banyak pengembang bekerja secara bersamaan pada bagian yang berbeda dari proyek tanpa saling mengganggu. Branch juga memudahkan rollback jika terjadi kesalahan pada suatu cabang, sehingga menjaga stabilitas proyek secara keseluruhan.
FORK
Fork dalam konteks Git dan GitHub adalah proses membuat salinan lengkap dari sebuah repositori yang ada ke akun GitHub Anda sendiri. Dengan melakukan fork, Anda mendapatkan kebebasan penuh untuk mengembangkan proyek tersebut secara mandiri tanpa memengaruhi repositori asli. Fork memungkinkan Anda melakukan perubahan, menambahkan fitur baru, atau memperbaiki bug di salinan repositori Anda, kemudian jika ingin mengusulkan perubahan tersebut ke proyek asli, Anda dapat mengirimkan pull request. Proses ini sangat berguna dalam pengembangan perangkat lunak sumber terbuka karena memungkinkan banyak kontributor bekerja secara paralel tanpa risiko merusak kode utama. Fork berbeda dengan clone, karena fork terjadi di server GitHub dengan membuat salinan repositori di akun Anda, sedangkan clone adalah menyalin repositori tersebut ke komputer lokal untuk pengembangan offline. Dengan fork, Anda juga bisa menyinkronkan perubahan dari repositori asli agar salinan Anda tetap up-to-date.
